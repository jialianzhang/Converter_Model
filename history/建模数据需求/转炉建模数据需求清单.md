# 转炉冶炼机理模型 · 建模数据需求清单

> **用途**：向钢厂申请数据，用于**修正标定 / 重构模型**以提高终点预测精度。
> **分类**：一、静态数据（每炉一个标量，吹炼前设定或吹炼后实测）；二、时序数据（吹炼过程中随时间变化）。
> **关键列——「当前状态」**：本模型 V1.0 已能运行，但精度受限于大量**假设值**与**缺失项**。本清单标注每项状态，指导优先申请：
> - 【已有实测】现数据集已含，继续提供即可；
> - 【模型假设】现用固定假设/经验值代替，**拿到实测可显著提精度**；
> - 【完全缺失】现完全无此项，**是精度天花板的主因，最高优先级**。
> **优先级**：★★★ 必需（无则模型跑不起来/无法标定）｜★★ 强烈建议（精度大幅提升）｜★ 可选（细化/研究用）。
> 示例值取自本项目 ~135 t 转炉、大废钢比工况的代表炉次与合理范围，供对表核对格式。

| 中间测温（时刻） | ℃, s | ★ | 倒炉时 | 温度曲线锚点；【完全缺失】 |

# 三、数据格式与提交建议

1. **两张表交付**：
   - `静态表`：每炉一行，列 = 上述一、各项字段（HeatID 主键）。可直接扩充现有 `data/raw/heats_template.csv`。
   - `时序表`：每炉多行，列 = HeatID + 相对开吹时间(s) + 各过程量/烟气分量；或每类时序一张子表。
2. **对齐基准**：统一以"开吹时刻 = 0 s"为相对时间零点；标注兑铁、开吹、停吹、出钢的绝对时刻便于追溯。
3. **单位统一**：温度℃、成分质量分数%、氧量 Nm³、流量 Nm³·min⁻¹、质量 kg 或 t（表内注明）。
4. **样本量**：静态标定建议 ≥100 炉同工艺窗口；若含烟气/枪位时序，30~50 炉即可显著提升动态精度。
5. **数据安全**：生产数据为企业专有，本项目已将 `data/raw/*.csv`、`data/rael/` 列入 .gitignore、永不入库；申请到的数据请置于 `data/raw/` 或 `data/rael/`。

# 四、拿到数据后模型能修正什么（对应关系速查）

| 拿到的数据 | 可修正 / 消除的模型假设或局限 |
|---|---|
| 终渣全组分 | 渣侧从"无校核"变可标定；修正 FeO 偏高、防跨窗口 FeO>CaO 倒挂；脱磷/脱锰渣侧闭环 |
| 烟气 CO/CO₂/流量 | 脱碳曲线、氧利用率、二次燃烧率 eta_post 由实测约束替代标定；终点碳自由预测精度上台阶 |
| 供氧+枪位时序 | 消除"恒流量+固定枪位"假设；混合能/乳化面积真实化；解废钢比温度残差、软吹制度 |
| 铁水带渣+前炉留渣 成分 | 修正初渣假设（MnO 8%等），消除初期 Mn 拔高/FeO 瞬态失真 |
| 辅料分批实际时刻 | 造渣进程、温度凹陷位置对准；石灰溶解动力学(粒径/活性度)标定 |
| 废钢分类+碳含量 | 废钢熔化从唯象抛物线升级为分块度/成分的机理熔化（张强式11熔点） |
| 炉龄/炉衬 | 散热、熔池几何随炉役漂移建模；MgO 衬蚀来源补全 |
| 副枪 TSC 过程点 | 开环→半闭环，突破温度散布物理地板 |

> 备注：本清单按当前 V1.0 模型结构反推整理；标注"完全缺失/模型假设"处即为精度提升空间最大处。数据到位后，重构优先级建议照第〇节梯队推进。
