# 转炉终点预测机理模型 V2 架构与逻辑升级方案

综合初版设计文档的工程结构、物理化学原理的硬伤纠错，以及迈向 SCI 级别高精度复现的修改建议，本方案对原设计进行了深度的提炼与重构。升级后的模型将真正实现“全流程动态耦合”，彻底解决参数静态化与逻辑割裂的问题。

## 一、 核心架构与模块重构 (Architecture & Modules)

为了支撑动态传质与精细化求解，模块划分需从原来的线性结构升级为**网状耦合结构**。

### 1. 新增与细化关键模块

- **混合能与动态传质模块 (重点新增)** 原设计中 $k_m, k_s$ 固化的缺陷，需通过新增 `mixing_energy.py` 和 `modified_mass_transfer.py` 解决。通过实时计算顶吹/底吹混合能 $\varepsilon$，动态更新 $k_m, k_s$，再转化为修正传质系数 $F_i$。
- **求解器模块独立** 将原有的单体 Solver 拆分为 `interface.py` (界面浓度计算)、`oxygen_balance.py` (氧平衡残差评估) 和 `root_solver.py` (寻根算法，如 Brent法/Newton法)。
- **热力学模块深化** 细化 `wagner.py` (钢液活度)、`ion_theory.py` (炉渣离子理论)、`equilibrium.py` (随温度变化的平衡常数)。*(注：若离子结构理论参数缺失，可在此模块预留光学碱度模型的 Fallback 方案)*。

### 2. 主循环顺序重排 (The 13-Step Main Loop)

原先的主模拟引擎逻辑较为粗糙，V2 版建议采用极其严格的单步时域离散（Euler-forward）顺序：

1. **更新混合能** (计算顶吹/底吹 $\varepsilon$)
2. **更新修正传质系数** (基于混合能和固相率更新所有 $F_i$)
3. **计算并更新表观平衡常数** $E_M$ (受当前温度和活度影响)
4. **求解界面氧活度** $a_O^*$ (调用寻根算法逼近 Residual=0)
5. **计算界面浓度** (代入 $a_O^*$ 求解各元素界面浓度)
6. **计算反应速率** ($\Delta m_i / \Delta t$)
7. **更新钢液相质量与成分**
8. **更新炉渣相质量与成分**
9. **更新石灰等造渣料的溶解状态**
10. **更新废钢的熔化状态**
11. **更新热量平衡** (统筹各项收支)
12. **更新温度** $T^{(l+1)} = T^{(l)} + \Delta T$
13. **更新热力学参数** (为下一时间步准备)

## 二、 致命公式纠错与逻辑澄清 (Formula Corrections & Clarifications)

### 1. 界面氧平衡方程（绝对致命错误修复）

原文档式(6)中的化学计量数存在严重错误，**必须严格修正**，否则整个耦合反应的物质守恒将被打破。

- **修改后公式 (Residual方程化)**：

  $$Residual(a_O^*) = 2J_{Si} + J_{Mn} + \mathbf{2.5}J_P + \mathbf{1.5}J_V + \mathbf{2}J_{Ti} + J_{Fe} + J_C - J_O = 0$$

  *(注：Mn为1，P生成*$P_2O_5$*为2.5，V生成*$V_2O_3$*为1.5，Ti生成*$TiO_2$*为2)*。

### 2. 公式属性与通用性澄清

- **式(3) 表观平衡常数** $E_M$：**不是常数！** 它是随界面成分、温度动态变化的中间变量，必须放在 Newton 求解器的迭代循环内部实时更新。
- **式(5) 脱碳逻辑**：$J_C = F_C(w[C]^b - w[C]^*) = G_{CO}(P_{CO}^* - 1)$ 是**速率平衡方程**，表明碳传质速率等于CO生成速率，不可将其与经验公式(16)混用。模型中应废弃经验公式(16)，并使 $G_{CO}$ 同样受混合能动态控制。
- **式(7) 界面浓度求解**：应抽象为针对 $C, Si, Mn, P, V, Ti, Fe$ 的通用表达式，而非仅针对 $Si$。

## 三、 数值求解器与传质动力学升级 (Solver & Kinetics)

### 1. 界面氧活度 $a_O^*$ 的鲁棒求解流

Newton-Raphson 法在处理分数幂次方程时极易出现负数导致崩溃。必须升级求解环：

```
[Inner Loop for Step 4 & 5]
Guess a_O^* (or use Brentq within [1e-8, 1.0] bounds to prevent negative values)
  -> Update current E_M
  -> Calculate all interface concentrations w([M])^* and w((MOn))^*
  -> Evaluate Oxygen Mass Balance Residual(a_O^*)
  -> Adjust a_O^*
Until |Residual| < tolerance
```

### 2. 传质系数 $F_i$ 的动态更新公式

废弃固定的 $k_m, k_s$。引入混合能 $\varepsilon$ 和固相率 $f_{solid}$ 概念：

- $k_m = k_{m}^0 \times (\varepsilon / \varepsilon_0)^n$
- $k_s = k_{s}^0 \times f(\text{stirring}) \times 10^{-n \cdot f_{solid}}$
- $F_i = \frac{\rho \cdot k}{100 \cdot M_i}$ （每个时间步重新计算 $F_C, F_{Si}, F_{Mn}, \dots$）

## 四、 标定策略与工程控制策略改进

### 1. 物理意义保留的参数标定

**不要直接暴力优化** $k_m, k_s$ **的数值**，这会破坏机理模型的物理意义。

- **建议策略**：以 Kitamura (2009) 提供的经典参数为基准，优化**修正乘子（**$\alpha$**）**。 目标变量：$\alpha_{steel}, \alpha_{slag}$ 关系式：$k_m^0 = \alpha_{steel} \times k_{Kitamura\_base}$ ； $k_s^0 = \alpha_{slag} \times k_{Kitamura\_base}$
- 同步优化 $G_{CO}^0$ (CO生成基准率) 和二次燃烧/热损失系数。

### 2. 仿真控制：自适应步长与终止条件

- **自适应时间步长 (**$\Delta t$**)**： 为了兼顾运算效率与极端情况下的微分方程稳定性，建议实施：
  - `1 s`：脱碳剧烈期（如吹炼中段，成分梯度极大）。
  - `2 s`：常规平稳冶炼期。
  - `5 s`：吹炼末期（碳氧反应减弱，成分趋于平缓）。
- **终止条件防早停**： 原逻辑 `C <= Target` 极易导致在吹炼初期（若废钢未化透或局部降碳过快）错误停机。 **修改为复合条件**：`达到目标吹氧量下限 AND 终点 C 满足目标范围`。
